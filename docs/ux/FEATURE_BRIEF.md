# Интерфейс списка покупок

## Направление

Глубина: `feature`. Это новый пользовательский сценарий поверх уже работающего LLM-сервиса.

Основная задача пользователя: назвать блюдо и за один проход получить список покупок на нужное количество людей. Результат для учебного задания: продемонстрировать цепочку «ввод → обработчик → вывод» с безопасным восстановлением после ошибок.

## Приоритеты

1. Один экран и одна основная кнопка.
2. Валидация до обращения к API.
3. Короткие безопасные сообщения вместо traceback и деталей провайдера.
4. Явное управление количеством людей и форматом ответа.
5. Удобство на ноутбуке и телефоне, включая длинный русский текст.

## Ограничения и допущения

- Используется существующий OpenAI-совместимый клиент и конфигурация через переменные окружения.
- Проверка выполняется с локальной Ollama, поэтому реальные ключи не нужны и не попадают в интерфейс.
- Аутентификация, история запросов и хранение результатов не входят в задание.
- Качество рецепта зависит от выбранной модели; интерфейс отвечает за управляемый сценарий и ошибки.
