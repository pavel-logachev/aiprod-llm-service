# Структура и потоки

## Структура экрана

1. Короткое объяснение результата.
2. Поле блюда и счётчик длины.
3. Выбор количества людей и формата.
4. Основная кнопка.
5. Область статуса и результата.

## Основной поток

Пользователь вводит блюдо, выбирает параметры, нажимает кнопку, видит индикатор загрузки и получает список. Повторный запрос выполняется через ту же форму без переходов между страницами.

## Альтернативные и восстановительные потоки

- Пустой или слишком длинный ввод: запрос к API не выполняется, фокус остаётся на форме, показана одна понятная фраза.
- Доступ не настроен: показано безопасное сообщение без ключа, URL и traceback.
- Таймаут или сеть: предложен повтор.
- Ошибка провайдера: предложено попробовать позже.
- Пустой ответ: предложено переформулировать блюдо.
